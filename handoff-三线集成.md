## 1. 三份契约

### 1.1 后端 → 前端：帧状态契约（已锁）
- `Cell { belief: float∈[0,1], type: honest|manipulative, standing: float∈[0,1] }`，逐帧 grid 快照 + 配置 + 显微镜判读记录。
- **关键性质**：契约与"搜到哪条规则"无关——规则换的是转移函数，不换渲染器画的东西。
- 字段精确定义、Python↔TS 一致性、验收清单 → 见 `frontend-backend-契约.md`。

### 1.2 前端 → 视频：高保真设计图 + 录屏素材契约（待锁）
前端是体验本体，视频拍的就是前端画面。两份交接物：

**A. 高保真设计图**（视觉真源，取代 design.md）
- 前端产出高保真设计图，视频组照此理解视觉皮肤、配色、版式、辉光质感。
- 视频组**不重新发明审美**，以设计图为准。

**B. 录屏素材契约**（前端最终交付给视频的镜头规格，需现在和视频组定死）
- 规格：分辨率 / 帧率 / 时长档（建议 1080p60，每镜头 8–15s）、是否要无 UI 的纯画布版。
- 镜头清单（shot list）：远景涌现群落 → 推近单只生物 → 点击触发显微镜判读 → 判"谎"的高光帧。
- 关键性质：**清单"与最终像素无关"**——按意图描述镜头。视频组先用设计图/mockup 剪出片子骨架（开场/转场/字幕/配乐/旁白），最后把真录屏换进去，**不重剪**。

### 1.3 三人 → 评委：pitch + 叙事脚本（已有）
- 一句话："一个会自己长出真话与谎言的宇宙——远看是涌现的生物在格子上厮杀，点开任意一只，LLM 宪法法院把它读成一个谎。"
- 叙事脚本三人共用，视频开场/字幕直接复用。

## 2. 四个集成节点（按相位，非绝对小时）

| # | 节点 | 谁参与 | 做什么 | 验收 |
|---|---|---|---|---|
| 1 | **T0 契约锁死会** | 三人 | 当场冻结三份契约（帧契约已锁 / 录屏素材契约 / pitch 脚本）。锁完各回各轴。 | 三份契约都有书面冻结版 |
| 2 | **24h 硬门 → 世界广播** | 后端→全员 | 发现冲刺结论（A 还是落 C）一出，立刻广播给视频："世界类型锁定 = ___"。此前视频只做与世界无关的部分（开场/品牌/字幕模板）。 | 视频组收到并确认世界类型 |
| 3 | **前后端接通**（48–60h） | 前端 + 后端 | 胜出规则插进引擎、前端接真帧。 | ✅ 真帧能渲染不崩 ✅ **关掉 LLM，CA 跑到底**（litmus） ✅ Python↔TS 字段对齐 |
| 4 | **前端 → 视频终交付**（60–72h） | 前端 + 视频 | 前端交一段干净录屏（对齐素材契约规格、无调试 UI、无鼠标乱晃）。视频塞进早剪好的骨架。 | 录屏符合素材契约；**此录屏同时是 demo 断网兜底** |

## 3. 防翻车 5 条铁律

1. **契约冻结**：T0 锁的三份契约，改动需三人都点头。帧契约"规则无关"，录屏契约也要做到"像素无关"。
2. **占位优先，真物后换**：前端对 stub、视频对高保真设计图/mockup，真资产晚换且不返工。
3. **绝不偷主轴时间**：后端/发现冲刺延期 → 砍它们自己范围，不吃前端主轴；视频延期 → 砍片长/转场，绝不逼前端提前交。
4. **单向依赖**：视频依赖前端输出，**永远不许反向**（别为"好拍"去改产品交互）。依赖成环 = 翻车。
5. **每个接通点一张验收清单**：接通不靠"感觉好了"，靠勾节点 3 / 4 的 smoke test。勾不满 = 没接通。
