# Veridia 前端(粒子球 / 手势)— 后端对接说明

> 交付物:`veridia-droplet.html`(单文件,Three.js + MediaPipe Hands,自包含)。
> 受众:后端工程师。目标:把这个前端从「JS 移植 demo」接到「真实后端数据」。
> 一句话:**对接只需一个开关 + 一份 trace。前端的所有可视化都是 Frame 的纯函数,后端不用为它改任何东西。**

---

## 0. 当前状态(必读)

- 前端**目前没有连后端**。为了能独立预览,我把后端 `ca/lenia.py` + `ca/rule.py` + `ca/morphology.py` 的算法**逐式移植成了 JS**,在浏览器里实时跑(同数学、同参数、同 seed 思路,但用的是 JS RNG,不是 numpy)。
- 这只是 demo 的数据源。**真实数据以后端为准**——见第 2 节,改一个开关即可切过去。
- 不变量沿用后端:**渲染层不 import 任何 LLM**;关掉 LLM 也能跑(litmus)。

## 1. 怎么本地跑

```bash
# 在仓库根目录
python3 -m http.server 8777
# 浏览器打开(摄像头/手势需要 localhost 或 https,file:// 拿不到摄像头权限):
#   http://localhost:8777/veridia-droplet.html
```

交互:挥手旋转 · 张手放大/握拳缩小 · 点击任意处→显微镜判读卡 · 右下面板可折叠(折叠后手势仍在跑)· 没摄像头可在面板切「鼠标」。

## 2. 唯一对接点 = Frame 契约

前后端**只**经一个数据结构耦合,就是逐帧的 `Frame`(定义见 `ca/state.py`、契约见 `frontend-backend-契约.md`):

| 字段 | 类型 | 含义 |
|---|---|---|
| `step` | int | 第几代 |
| `width` / `height` | int | 网格尺寸(当前 48 × 48) |
| `belief` | float[ W*H ] ∈[0,1] | 信念场;1=真理极,0=谎言极。**行优先 `i = y*width + x`** |
| `type` | int[ W*H ] | 阵营:`0=honest(晶族)`、`1=manipulative(雾族)`。**出生即定,永不变** |
| `standing` | float[ W*H ] ∈[0,1] | 声望/可信度 |

顶层 trace(`python -m ca.run` 产出,落在 `web/src/ca/fixtures/run-trace.json`):

```jsonc
{ "game_id":"veridia-ca-7", "codename":"veridia-ca",
  "config":{ "width":48,"height":48,"steps":80,"stride":2, ... },
  "frames":[ {step,width,height,belief[],type[],standing[]}, ... ],
  "microscope":[ {step,x,y,claim,verdict,reason}, ... ] }
```

> 前端只需要 `frames[]`。`microscope[]` 可选(前端点击判读目前用本地 `template_record` 同逻辑算,接 LLM 真判读后可改用这里的记录)。

## 3. 接入方式(改一个开关)

我在 `veridia-droplet.html` 里预留了对接钩子,搜索 **`★ 后端对接点`**:

```js
const USE_REAL_TRACE = false;   // ← 改成 true
const TRACE_URL = 'web/src/ca/fixtures/run-trace.json';
```

把 `USE_REAL_TRACE` 改成 `true`,前端就会 `fetch(TRACE_URL)`、逐帧调用 `applyFrame()` 把 `belief/type/standing` 喂进渲染。**改完即生效,其它一律不用动。**

### 两条路任选

- **A. 静态回放(最快,推荐先跑通)**:后端 `python -m ca.run` 生成 `run-trace.json`(已在仓库里有一份样例,41 帧),前端直接读。换 seed / 改步数 → 重跑生成新 JSON 即可。
- **B. 实时流(可选,进阶)**:后端起一个 WebSocket/SSE,每代推一个 Frame JSON。前端把 `applyFrame(frame)` 接到消息回调上即可(`applyFrame` 接收的就是一个 `{step,width,height,belief,type,standing}`)。

> ⚠️ 真实 trace 的 `type` 数组(后端 numpy seed)和 demo 的 JS seed 不一样,所以 `applyFrame` 里会同步刷新粒子的阵营颜色(已写好,无需关心)。

## 4. 前端怎么把 Frame 画出来(后端无需实现,仅供理解)

每颗粒子 = 一个 cell,**三通道分开编码**:

- `type` → **色相**:晶族=金、雾族=深蓝(出生固定,不会互相转化)。
- `belief` → **亮度 / 发白**:越靠真理极越亮、趋近 1 时发灼白;跌破 0.5 阈值→变暗成尘埃、在形态学里"熄灭"。
- `standing` → **聚合度 / 光晕稳定**。

派生可视化(全部前端按后端同算法本地算):
- **晶族/雾族点亮面积**(左下拔河条):`belief>0.5` 且按 type 分组计数 → 看两族争"还亮着的面积"。
- **生物数 / 混沌边缘**:`morphology.py` 的 8-连通连通域 + 逐帧 `mean|Δbelief|`。
- **显微镜判读卡**:点任意格 → claim / verdict / reason + 实时博弈条(信念、声望、邻里阵营、邻里点亮)。verdict 方向是**机械事实**(由 type 定:晶族=真、雾族=谎),不取 LLM。

## 5. 几条必须知道的约定(避免对接踩坑)

1. **行优先索引**:`i = y*width + x`,和后端 `Frame.idx` 一致。
2. **type 永不变**:细胞不会"从蓝变金"。蓝族只会 belief→0 **变暗熄灭消失**;金族 belief→1 **变亮发白**。主题即:*真相不是战胜谎言,是谎言自我熄灭*。别把它实现成"阵营转化"。
3. **网格是环面(torus)**:后端是一张 wrap-around 的 48×48 平面,生物滑出一边从另一边回来。**"映射到球面"纯属前端可视化选择**,后端不用管几何。
4. **初始态由后端定义**:`ca/lenia.seed_orbium` —— 在 (W/4, H/4) 放**一只 Orbium 生物**,其余≈0 + 微噪声;type 全场随机 ~25% 雾族;standing 全场 0.5。所以开局是"黑暗尘埃球 + 一处亮斑",不是均匀铺满。前端照 Frame 渲,后端怎么定初值,画面就怎么起。
5. **litmus**:渲染层不依赖 LLM;`belief/type/standing` 的演化全在后端 CA,LLM 只在显微镜(read-only)。前端不碰 LLM。
6. **数值范围**:`belief/standing ∈ [0,1]`,`type ∈ {0,1}`,`len(belief)==len(type)==len(standing)==width*height`。

## 6. 你(后端)需要做的 / 不需要做的

- ✅ **需要**:产出可被前端读到的数据——要么把 `run-trace.json` 放进 `web/src/ca/fixtures/`(静态回放),要么起一个推 Frame 的接口(实时流)。
- ❌ **不需要**:为这个前端改 CA 引擎 / 规则 / 契约。契约已固定,前端跟随。
- 如果你升级了规则(转移函数),只要 `.frame()` 仍吐同形状的 Frame,前端**一行都不用改**(契约文档原话:"规则换的是转移函数,不换渲染器画的东西")。
