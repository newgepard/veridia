# Veridia 视觉设计方向报告
### 「真相 vs 谎言」元胞自动机 — 从「丑」到「天文台级控制台」的可落地蓝图

> 来源:本会话中 design-research 子 agent 的研究产出(扒 Sandspiel / Lenia-CE / tixy / particle-life / Universe Sandbox / Star Citizen mobiGlas / scificn-ui / Leva / tldraw 等真实源码与权威文章)。
> **⚠️ 注意:本报告走的是「科幻天文台 + Orbitron」方向,与已锁定的《光遇》×《我的世界》方向(见 `design.md`)是两套并存的参考素材 —— 配色 / 字体体系不同,对比取用,不互相覆盖。**

> 一句话定调:**让会动的发光生物当唯一主角,UI 退成悬浮在活宇宙之上的冷静仪表罩。** 暗场 + 逐像素色阶让生物自发光,等宽数据读出 + 克制霓虹 + 玻璃 HUD 给它「创世天文台」的庄严。下面所有 hex / 字体 / 代码思路都从真实源码或权威文章扒来,可直接进 `tokens.css` / `theme.ts`。

> 研究者校准:多个 ALife 标杆其实是**浅底实验室风**(Sandspiel、官方 Lenia),只有 Complexity Explorables 版 Lenia 是暗场发光。Veridia 的「真谎 + 神学」主题该走**暗场**,故以「暗场 + per-pixel colormap」为主轴,浅底参考只取 canvas-as-hero 与工具自染手法。

**这一类前端「为什么好看」的共性(提炼):**
1. **Canvas-as-hero**:画布满视口/居中统治,chrome 退到角落浮动。
2. **暗场 + per-pixel colormap 而非加 glow**:生物发光的真相是「色阶映射到每像素 fillStyle 铺在暗场」——深靛/近黑作零值,黄/品红作密集核心高光。
3. **极简 chrome、双色或单强调色**,克制到「靠纯饱和色在纯背景上自己亮」。
4. **等宽字体 + 实时数据读出**:等宽=机器在自组织的视觉签名,精准契合 Veridia「创世立法后机械运行」的自然神论。
5. **颜色即图例即连接线**:工具按钮用绘制物本色自染,同色贯穿「像素↔按钮↔面板」。
6. **约束/状态做成颜色事件**:染蓝、alpha 渐隐、冷→暖拖尾,状态被「看见」而非读文字。

## B. 给 Veridia 的视觉方向

**整体 vibe 一句话:** 一座漂浮在深空里的**创世天文台**——你透过冷静的玻璃仪表,俯瞰一锅被法则点燃、自行涌现出信真暖光与信谎冷暗的活宇宙。庄严、理性、有机,但绝不甜美卡通。

### 配色板(可直接进 `tokens.css`;已映射真/谎 + 宇宙/创世 + 涌现生命)

```css
/* ── 深空 / 创世底 (背景,绝不用纯黑) ── */
--void-0:        #070812;   /* 最底:近黑带蓝,深空虚无 */
--void-1:        #0B1026;   /* 次层 / 玻璃面板底色:深空蓝 */
--surface-up:    #131A33;   /* 更高的浮层(越高越亮,靠亮度表海拔不靠投影) */
--grid-line:     #22304D;   /* 创世坐标网格 / 薄边框,冷蓝灰,克制到几乎看不见 */

/* ── 信真 · 暖 (真理之光 / 涌现生命) ── */
--truth-core:    #F7C948;   /* 琥珀金:信真核心、status 暖、笔刷 */
--truth-glow:    #FFE9B0;   /* 暖白高光:生物自发光最亮处 */

/* ── 信谎 · 冷暗 (死寂 / 谎言区) ── */
--lie-mid:       #273043;   /* 墨蓝灰:信念场冷区 */
--lie-deep:      #1A2030;   /* 谎言最深处,近背景 */

/* ── 操纵蓝 (警示,被操纵格) ── */
--manip-1:       #5468FF;   /* 电蓝(染色基色) */
--manip-2:       #00B8FF;   /* 更刺眼的告警蓝(辉光外圈) */

/* ── UI 霓虹点缀 (克制) ── */
--accent-cyan:   #00C1D4;   /* 边框 hover / 关键交互的克制霓虹 */
--text-hi:       #E6ECFF;   /* 主文字:带蓝的近白,不纯白 */
--text-lo:       #8A97B8;   /* 次级文字 / 标签 */
```

> 语义逻辑:**暖给「信真/生命」,冷蓝给「理性/操纵」,深空蓝黑当裁判**。belief 场推荐用一条**暖↔冷散色阶(diverging)**而非 viridis——因为你要语义(真暖/谎冷),但渲染机制(per-pixel on dark)照搬 Lenia-CE。色阶端点:暖端 `#FFE9B0`→`#F7C948`,中性 `#273043`,冷端 `#1A2030`;操纵覆盖层单独叠 `#5468FF`。
> 现成可抄参考板:media.io「Cyberpunk」一套里同时有暖珊瑚/琥珀 + 冷青 + 近黑(https://www.media.io/color-palette/cyberpunk-color-palette.html);「Deep Space」给了深空黑 + 靛蓝 UI + 琥珀三件(https://www.media.io/color-palette/space-color-palette.html)。

### 字体三件套(全部 Google Fonts SIL OFL 可商用,优先 fontsource 自托管)

> ⚠️ **与锁定方向的差异**:`design.md` 已锁 **Cinzel(碑文)+ Cardo(经卷正文)+ Monocraft(像素数据)**,主打《圣经》庄严 + 《我的世界》体素。本报告的 Orbitron 三件套是另一条「科幻天文台」路线,留作对比,**不覆盖锁定项**。

### 动效与质感(全部克制,opacity 0.04–0.18,绝不抢生物)
- **生物自发光**:canvas 画发光层时 `ctx.globalCompositeOperation = 'lighter'`(加色),配半透明径向渐变软点,重叠处自动叠亮成光团 = 涌现生命的暖光(MDN: https://developer.mozilla.org/en-US/docs/Web/API/CanvasRenderingContext2D/globalCompositeOperation)。用完 `ctx.restore()`。
- **标题/判读结果辉光**:多层 `text-shadow`(内核 `#FFE9B0`→外圈 `#F7C948`),只给标题和真谎判读,**正文别加会糊**(CSS-Tricks: https://css-tricks.com/how-to-create-neon-text-with-css/)。
- **操纵告警边框**:`box-shadow` 多层 + inset 电蓝辉光(Coder's Block 原则「first on top, last on bottom」:https://codersblock.com/blog/creating-glow-effects-with-css/)。
- **玻璃 HUD**:`background: rgba(11,16,38,.65) + backdrop-filter: blur(16px)`(12–20px 甜区)+ `1px` 冷蓝描边 + 圆角 16px。后面必须有 canvas 复杂度才像真玻璃(StudioLimb: https://www.studiolimb.com/guides/glassmorphism-css-tutorial.html)。
- **创世坐标网格**:双向 `repeating-linear-gradient` 冷蓝灰细线 40px 一格,暗到几乎看不见。
- **观测设备氛围**:极淡扫描线(`repeating-linear-gradient` 每 3px 一条 `rgba(0,0,0,.18)`)+ SVG `feTurbulence` 噪点 opacity 0.04(CSS-Tricks Grainy Gradients: https://css-tricks.com/grainy-gradients/)。把屏幕变成「取景器」,但只做底噪。
- **暗色铁律**(Refactoring UI 系):背景**绝不纯黑**(`#000` 投不了阴影、表达不了海拔);**层级靠「越高的面越亮」**;亮色**大面积时降饱和**(否则炸眼),只在小发光点用满饱和;灰要**往主色偏蓝**别死灰。

## C. 具体排版方案(canvas 当英雄主角)

**核心架构转向(治本):** 抛弃「720px 居中垂直堆叠」。改成 **canvas 全屏铺底(`position: fixed; inset: 0`)+ 一层 `absolute` 定位的 HUD chrome 浮在上面**。这正是 tldraw 的分层模型:跟相机走的 `OnTheCanvas`(选中框、细胞标记)vs 屏幕固定的 `InFrontOfTheCanvas`(控制条、显微镜、HUD 角标)。你现在病根就是没分这两层。


**各区落点(从研究里收敛):**
- **Canvas**:`position: fixed; inset: 0` 占满,`image-rendering: pixelated`(放大脆,Sandspiel),逐像素散色阶渲 belief 场 + `'lighter'` 自发光层。它是唯一主角。
- **模式切换(回放 trace ↔ 实时模拟)**:左上汉堡 + 右上小徽章(Universe Sandbox 汉堡收口)。回放模式底部条变成时间轴 scrubber;实时模式底部条放速度/画笔。
- **笔刷/工具(实时)**:竖列浮左,**用绘制物本色自染**(Sandspiel),选中态把该色饱和度乘上去——调色板即图例,不另画色卡。
- **显微镜面板**:点细胞才从右侧滑出的玻璃面板(Universe Sandbox 点选天体才出属性)。结构:坐标 → 它说的话(等宽框)→ 真/谎判读 Badge(`TRUTH`/`LIE`,scificn-ui)→ belief 值条 → 被谁影响。
- **底部控制条**:`⏮ ⏯ ⏭` + 速度滑块 + **live 数据读出**(gen 代际 / truth% / lie% / 操纵格数,copy.sh 仪表盘 + Universe Sandbox 时间组),JetBrains Mono 等宽对齐。
- **研究统计 / 数据叠加**:做一个图标 toolbar 切「信念热力 / 边界 / 操纵分布」叠加层(Cities: Skylines info views),或单独玻璃卡浮右上。若走「可逛画廊」路线,trace 列表用 Complexity Explorables 纵向列表(缩略图 + 标题 + 一句副标)。
- **HUD 叠层组织**:全部 `InFrontOfTheCanvas`(屏幕固定,不随缩放变大),`z-index` 分层,pointer 事件标记已处理避免误触画布(tldraw)。面板用切角缺口 + 薄冷蓝边(scificn-ui / Star Citizen 几何直线)。

> 想省力:`npm i leva` 直接把「模拟参数(规则/速度/真理浓度/种子)」接管成右侧悬浮面板,几小时拿到专业控制台手感。scificn-ui 同栈可直接抄 Panel/Badge/Button。

## D. 按钮 / 交互状态(治「假按钮」)

现在按钮像假的,是因为**零状态反馈**。每个可交互元素都要走完整状态机,且**用颜色/辉光/位移给即时物理反馈**:

| 状态 | 视觉规格(以主操作按钮为例) |
|---|---|
| **default(静息)** | `--void-1` 底 + `1px solid --grid-line` + `--text-hi` 字;切角缺口(非圆胖);JetBrains/Space Grotesk |
| **hover** | 边框转 `--accent-cyan` + 轻 `box-shadow: 0 0 8px rgba(0,193,212,.5)`(克制辉光)+ 底色亮一档;`transition: .15s`,光标 `pointer` |
| **active / 按下** | `transform: translateY(1px) scale(.98)` + inset 暗影,给「真被按下去」的物理感;辉光瞬间增强再回落 |
| **toggled-on(选中/笔刷激活)** | 用该工具语义色填充(TRUTH→`--truth-core` / LIE→`--lie-mid` / MANIP→`--manip-1`)+ 持续柔辉光,和未选明显拉开(Sandspiel 选中硬反差) |
| **playing(模拟运行中高亮)** | ⏯ 按钮持续呼吸辉光(`@keyframes` text-shadow 强度 0.6↔1.0 循环)+ 顶部一条极细进度/活动指示,让人确信「它在跑」(对应自然神论「神缺席后世界自行运转」) |
| **disabled** | `opacity: .4` + 去掉辉光 + `cursor: not-allowed` + 边框降到 `--lie-deep`,明确「现在不可点」(如实时模式下 trace scrubber 禁用) |
| **loading / 未来态** | 预留:边框跑一圈扫描光(`conic-gradient` 旋转)或 Badge 切 `SCANNING` 态(scificn-ui),给 trace 加载/模拟初始化用 |
| **状态徽章(判读)** | `TRUTH`=暖金底深字 / `LIE`=冷青描边 / `MANIPULATED`=电蓝 + 告警辉光(scificn-ui Badge 体系) |

**全局交互原则:** 每次点击/悬停都要有「看得见的回应」(颜色 + 辉光 + ≤2px 位移 + ≤150ms 过渡);状态优先做成**颜色事件**(被操纵→染电蓝、信念翻转→暖冷渐变),呼应 tixy「约束被感觉到而非读到」。

## E. 明确要避开什么

- ❌ **generic Bootstrap / Material / Ant 默认皮肤**——圆胖按钮、蓝色 primary、卡片阴影、紫色渐变 banner,一眼「模板站」,彻底破坏天文台庄严。
- ❌ **默认 HTML 控件**(裸 `<button>`/`<input>`/`<select>`)——这正是当前「假按钮」的来源,必须全部换成有状态机的自定义组件。
- ❌ **720px 居中 + 垂直堆叠**——canvas 必须全屏当主角,UI 浮在其上,不是把画布塞进一个窄文档流。
- ❌ **纯黑 `#000000` 背景**——投不了阴影、表达不了海拔,死。用 `#070812`/`#0B1026`。
- ❌ **大面积高饱和糖果色 / 甜美卡通**——和「锋利/庄严/创世」内核冲突;亮色大面积要降饱和,满饱和只留给小发光点。
- ❌ **满屏 glow / 扫描线拉满**——廉价化。辉光只给信真核心 + 操纵告警 + 标题判读;质感层 opacity 压在 0.04–0.18。
- ❌ **多套字体大杂烩 / Orbitron 当正文**——锁死三件套各司其职。
- ❌ **静态无反馈交互**——任何点击/悬停都要有颜色或辉光回应,否则又变「假按钮」。
