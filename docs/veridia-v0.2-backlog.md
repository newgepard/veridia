# Veridia v0.2 Backlog —— 字段回补清单(按 ROI 排序)

> Q2 已锁最小持久字段 `{ type, standing }`(+ 每帧 ≤4 态输出 `{signal, true_action}`)。
> 以下是"加回来 ROI 最高"的候选,**不进 MVP**,跑出来发现单薄时按顺序补。

## 🥇 `belief`(信念 / 正在传播的 claim)
- **解锁**:谎=会传染的错误信念;"真相波 vs 谣言波"在格子上竞争扩散 → 把游戏从"带信号的收益博弈"升格成**信息的流行病学**。
- **成本**:字段极便宜(1 值),贵在"信念更新规则"。
- **注**:这其实是"换种"而非加料——它让模拟**真正关于信息**。很可能不是 backlog 项,而是 MVP 的核(见正在拷问的 Q2 A/B 分叉)。

## 🥈 `reputation` / `score` 拆分(财富 vs 信任)
- **解锁**:把 MVP 合并的 `standing` 拆开 → "又富又越来越不被信"的高明骗子,欺骗戏剧的心脏。
- **成本**:1 个标量 + 规则微调。**最便宜的"加深主题"。**
- **触发**:跑出来发现骗子又富又横、demo 单薄 → 立刻拆。

## 🥉 `commitment`(可验证承诺)
- **解锁**:诚实细胞立公开可验证承诺,毁约=铁证谎;操纵者承诺没人信 → **透明从弱点变武器**,诚实能主动赢。
- **成本**:中(pledge 字段 + 校验)。
- **触发**:想回答"诚实凭什么赢",或想要"诚实领地反攻"的戏剧。

## 低 ROI,MVP 明确不加
per-edge 记忆(贵,standing 已近似)· age/冷却(救赎动态,锦上添花)· 影响半径 · 能量代谢(偏离真/谎主题)。
